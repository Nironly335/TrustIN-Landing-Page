Здесь разрабатывается landing page для проекта TrustIN.
Что такое TrustIN
TrustIN — сервис для легальной самозанятости мигрантов в Германии. Помогает зарегистрироваться как Freiberufler, найти учеников для репетиторства и автоматически выставлять официальные счета (Rechnung).
Продукт находится на стадии до MVP. Команда собирает вейтлист перед запуском в Саксонии.
Цель лендинга
Один результат: человек оставляет email и имя в форме вейтлиста.
Никаких других целей у этой страницы нет.
Откуда придёт трафик
→  Украинские Telegram-каналы в Дрездене и Лейпциге (2,000–10,000 участников)
→  Русскоязычные Facebook-группы мигрантов в Саксонии
→  Личный аутрич основателя — ссылка в переписке
→  Позже: SEO и платные объявления
⚠️  Большинство пользователей придут с телефона. Мобильная версия — приоритет номер один.
Основная аудитория — мигранты
→  Украинцы с визой §24 в Дрездене, Лейпциге, Хемнице
→  Иностранные студенты TU Dresden и Uni Leipzig (§16b)
→  Русскоязычные мигранты в Саксонии
→  Возраст: 22–38 лет
→  Уже преподают репетиторство — неофициально, за наличные
→  Или хотят начать, но не знают как
⚠️  Боль острая: бюрократия непонятна, язык сложный, ситуация срочная.
Вторичная аудитория — немецкие репетиторы
→  Студенты и выпускники немецких университетов
→  Уже дают уроки неофициально
→  Боль другая: не виза, а Schwarzarbeit-риск и бумажная волокита
⚠️  Немецкая версия лендинга — отдельный посыл. Без визового контекста.
Блок 1 — Hero
ПРИОРИТЕТ: КРИТИЧЕСКИЙ
Что делает:
Первое что видит пользователь. Должен узнать себя за 3 секунды и нажать кнопку.
Элементы:
→  Навбар: логотип TrustIN слева + переключатель языков справа (RU / UK / EN / DE)
→  Заголовок H1 — крупно, одна мысль
→  Подзаголовок — 2-3 строки, конкретно
→  Поле email + кнопка «Записаться»
→  Микрокопи под кнопкой: «Бесплатно · Без юриста · На твоём языке»
⚠️  Никакого меню, никаких ссылок в навбаре кроме переключателя языков. Только логотип и языки.
Блок 2 — Узнай себя
ПРИОРИТЕТ: ВЫСОКИЙ
Что делает:
Пользователь читает и думает «это про меня». 3 строки максимум.
Формат:
Три пункта с иконкой или маркером. Не буллеты — короткие утверждения.
Блок 3 — Как это работает
ПРИОРИТЕТ: ВЫСОКИЙ
Что делает:
Объясняет продукт за 30 секунд. Три шага.
Формат:
Три карточки горизонтально (на десктопе) / вертикально (на мобиле). Номер шага + заголовок + 2 строки текста.
Блок 4 — Интерактивный выбор пути
ПРИОРИТЕТ: СРЕДНИЙ — можно добавить в версии 1.1
Что делает:
Пользователь выбирает кто он — и видит релевантный контент. Два варианта:
→  «Я уже даю уроки» — контент про легализацию
→  «Я хочу начать» — контент про старт с нуля
Механика:
Два таба или две большие кнопки. Контент блока меняется при переключении. Реализуется на JS без перезагрузки страницы.
⚠️  Если времени мало — этот блок можно убрать из версии 1.0 и добавить позже.
Блок 5 — Форма вейтлиста
ПРИОРИТЕТ: КРИТИЧЕСКИЙ
Поля формы:
→  Имя (text input, обязательное)
→  Email (email input, обязательное)
→  Кто ты? (radio или select: «Репетитор-мигрант» / «Хочу начать преподавать» / «Ищу репетитора» / «Немецкий фрилансер»)
→  Город (select: Дрезден / Лейпциг / Хемниц / Другой)
→  GDPR-чекбокс (обязательный): «Я согласен на обработку персональных данных согласно Privacy Policy»
→  Кнопка «Записаться в вейтлист»
После отправки:
Форма скрывается. Появляется сообщение: «Ты в списке. Мы напишем когда запустимся в твоём городе.»
Куда идут данные:
Google Sheets через Google Apps Script или Airtable — уточнить у заказчика перед началом работы.
⚠️  Кнопка задизейблена пока не заполнены имя, email и не поставлен GDPR-чекбокс.

7. Технические требования
Стек
→  HTML + CSS + Vanilla JS — никаких фреймворков
→  Один файл index.html (CSS и JS встроены или в отдельных файлах рядом)
→  Google Fonts через @import
→  Никаких внешних JS-библиотек кроме допустимых (уточнить с заказчиком)
Мобильная адаптация
Mobile-first. Основная аудитория — пользователи телефона пришедшие из Telegram.

Форма и данные
Перед началом работы уточнить у заказчика куда идут данные:
→  Вариант А: Google Sheets через Google Apps Script (бесплатно, просто)
→  Вариант Б: Airtable через API (чуть сложнее, удобнее для сегментации)
→  Вариант В: Заглушка — данные нигде не сохраняются, только UX отработать

Производительность

→  Размер страницы без шрифтов: до 100kb
→  Шрифты загружать с display=swap
→  Изображений нет — только CSS и текст
GDPR
→  Чекбокс согласия обязателен перед отправкой
→  Privacy Policy страница или модальное окно — заглушка с базовым текстом
→  Impressum — заглушка (заказчик предоставит реквизиты позже)


8. Прототипы для референса
В проекте уже есть два прототипа лендинга. Используй их как референс, НЕ как финальный дизайн.
trustin_v6_form_middle.html — основа
Этот прототип берём за основу. В нём:
→  Правильная структура цветов (всё через CSS-переменные, работает без внешних зависимостей)
→  Полная форма с GDPR-чекбоксом и состоянием успеха
→  Сплит-hero 50/50 — можно упростить до центрированного если нужно
→  Три языка реализованы (EN / DE / RU) — добавить UK
trustin_landing_prototype.html — за типографику
Из этого прототипа взять:
→  Шрифтовую пару: Fraunces + DM Sans (или Inter)
→  Анимацию success-сообщения формы
→  Стиль proof-секции со статистикой
⚠️  Этот прототип сломан без внешних CSS-переменных — не запускать как есть.
